# シンプルコメビュ（使い方）

ニコ生のコメント・運営通知・ギフトなどをまとめて表示し、読み上げ・OBS表示・個人演出までできるローカルアプリです。

通常のコメビュが捨てがちな NDGR の各種メッセージも一覧・保存します。

---

## できること

- 放送に接続して、コメントをリアルタイム表示
- 過去コメントの全件取得
- 生IDごとのスキン・フォント・VOICEVOX 設定
- 広告・ギフト・来場などのイベント種別ごとの SE と表示
- OBS への横流しコメント（スキン）とリスト表示
- コメント投稿（184 可）
- 任意で: AI 返信、タグ変更、YouTube 受付、リアルタイム字幕、RVC

---

## 前提

配信前に、使う機能だけ用意すれば足ります。

| 機能 | 必要なもの |
|------|------------|
| コメント表示・保存だけ | このアプリ |
| 読み上げ | [VOICEVOX](https://voicevox.hiroshiba.jp/) エンジン（既定 `http://127.0.0.1:50021`） |
| OBS 表示 | OBS Studio + **WebSocket サーバ**（既定 `ws://127.0.0.1:4455`） |
| リアルタイム字幕 | RTFW（別途起動） |
| RVC | MMVCServerSIO。LAN利用時の音声制御サービス／ワーカーは本リポジトリに同梱 |
| AI 返信 | Codex など、設定した実行環境 |
| タグ変更・YouTube 受付 | ログイン済み Chrome プロファイル（Selenium） |

## 起動

Python 3.11以上とGitをインストールし、初回だけ次を実行します。

```bat
setup.cmd
```

Python Launcherを使わない環境では、`PYTHON_EXE` にPython 3.11以上の実行ファイルを指定してから実行できます。

依存関係は `requirements.txt` から、このフォルダ直下の `.venv` へ導入されます。

### 普段使うとき（おすすめ）

```bat
start.cmd
```

- GUI を起動します
- 介入 API（`http://127.0.0.1:8793`）も一緒に立ちます
- コンソールは最小化されます

コンソールを見て起動確認したい場合:

```bat
start.cmd --wait
```

### ウィンドウだけ静かに起動したいとき

```bat
start_silent.vbs
```

GUI のみです。介入 API や RTFW は付きません。

### 開発用の直接起動

```powershell
.venv\Scripts\python.exe main.py
```

入口の指定例:

```powershell
.venv\Scripts\python.exe main.py --entrypoint gui
.venv\Scripts\python.exe main.py --entrypoint api
```

## 最初の一回だけ

### 1. 基本設定を保存する

タブ **基本設定** → サブタブ **基本応答**

1. 「基本設定で読み上げる」を好みで ON/OFF
2. VOICEVOX が動いていることを確認し、「VOICEVOX話者を再読込」
3. 基本話者・音量・基本スキン・フォントを決める
4. **保存**

同じタブの **再生速度** で、コメントが溜まったときの読み上げ倍率を調整できます。

- 待機 0 件の倍率
- 待機 1 件時の倍率
- 最大倍率

### 2. OBS に載せる

タブ **OBS連携**

1. OBS の WebSocket を有効にし、URL / パスワードを入力
2. **接続テスト**
3. **ソース一覧** で Browser Source 名を合わせる
4. 既定の行は次のイメージです

| 名前 | 用途 | URL |
|------|------|-----|
| 右から左スキン | 横に流れるコメント | `http://127.0.0.1:8792/` |
| 通常リスト | 下に積むリスト | `http://127.0.0.1:8792/list` |
| リアルタイム字幕 | 音声の字幕 | `http://127.0.0.1:8788/overlay` |

5. **一括更新**（起動時にも一度走ります）

OBS 側では Browser Source の背景を透明にしてください。

### 3. リスト見た目を整える（任意）

**基本設定 → 通常リスト**

背景・アイコン・色・最大行数などを変更します。  
ここは変更すると自動保存され、OBS のリスト表示へすぐ反映されます。

## 普段の流れ（配信中）

### コメントタブ

1. **コメント** タブを開く
2. 放送欄に `lv...` または watch URL を入れる
3. **接続** でリアルタイム受信
4. 必要なら **全件取得** で過去分も取る
5. 止めたいときは **停止**

#### ページ上のスイッチ

| 項目 | 意味 |
|------|------|
| 読み上げ | この放送で VOICEVOX 読み上げするか |
| OBS | この放送を OBS オーバーレイへ出すか |
| 文字起こし紐づけ | リアルタイム字幕の結果をこの放送の CSV に紐づける |
| コメントをベクター化… | ON の間だけ embedding / 索引更新 |
| ログ | INFO / DEBUG / TRACE など |
| TRACEで各メッセージもログ | 1 件ごとの詳細（重い） |

#### 複数放送

- タブ右の **+** で放送ページを追加（最大 20）
- タブの × で閉じる
- 別放送を同時に見たいときに使う

#### コメント投稿

下の入力欄から送信できます。

- **184で投稿** で匿名投稿
- 接続中でないと送れない場合があります

#### 表の見方

主な列:

- アイコン / 名前 / No / 本文
- 状態 / ユーザーID / コマンド / 投稿時刻
- 種別（コメント・広告・ギフトなど）

下半分は raw とログです。

右クリックなどから、個人設定・履歴・AI 返信・自動演出などが使えます（環境によってメニューが増えます）。

### 取得履歴タブ

過去に触った放送の一覧です。

1. 放送者・タイトル・期間で絞る
2. 行を選ぶ
3. **現在タブへ入力** / **接続** / **全件取得**

ダブルクリックでも現在タブへ lv を入れられます。

## 演出の決め方（三層）

コメントが来たときの見た目・声は、だいたい次の順で決まります。

```text
アカウントID設定（その人専用）
        ↓ なければ
イベント設定（広告・ギフトなどの種別）
        ↓ なければ
基本設定（全体の既定）
```

### アカウントID設定

生IDごとに:

- 有効 / 読み上げ / スキン出力 / リスト出力
- 表示名・名前ロック
- スキン・フォント・色・VOICEVOX

表を編集すると DB に保存されます。  
「この人だけ別ボイス」「常連だけスキン」などをここでやります。

コメント番号から設定したい場合は、介入 API 経由の個人設定スキルも使えます（後述）。

### イベント設定

種別ごとに:

- 有効
- 固定音声（SE）
- 表示テンプレート（例: `【ギフト】{advertiser_name}さんが…`）
- スキン / フォント / VOICEVOX

初回は **全イベント初期設定** でひな形を入れてから直すと楽です。

運営コメント・広告・ギフトなどは、ここで SE と文言を決める想定です。  
SE と読み上げは同時に走ることがあります（SE 終了待ちで詰まらせないため）。

## オプション機能

### AI 返信

**基本設定 → AI返信**

1. 「AI返信フックを使う」を ON
2. ルールを 1 行 1 件で書く（例: `おはよう=>おはようございます`）
3. または接頭辞（既定 `>AI`）で Codex 呼び出し
4. 保存

### タグ変更

**基本設定 → タグ変更**

1. 有効化
2. ルール例: `タグ変えて=>雑談,ゲーム,初見歓迎`
3. Chrome アカウントを選ぶ
4. 保存

コメントにキーワードが含まれると、放送ページのタグ UI を自動操作します。

### YouTube 受付

**基本設定 → YouTube受付**

有効中、コメントに流れた **最初の 1 本だけ** YouTube を Chrome で開きます。  
次を受け付けるときはコメントページの **YouTube受付リセット**。

### リアルタイム字幕

1. **リアルタイム字幕** タブで API / オーバーレイ URL を確認
2. マイクまたは PC 音声を開始
3. OBS の字幕ソースが `http://127.0.0.1:8788/overlay` を見ていること
4. コメント側で保存したい放送は **文字起こし紐づけ** を ON

### RVC

**RVC** タブで接続先、MMVC本体、音声制御サービス、起動Python、トークン設定、モデル、音声デバイスを指定し、**RVCを使用** で ON。
コメント読み上げとは別系統です。

固定のドライブ、IPアドレス、Python、MMVC配置先、仮想音声デバイス名は前提にしていません。各項目は利用するPCに合わせて指定します。

#### このPCのMMVCへ直接接続

1. 接続先を **メインPC（このPC）** にする
2. `MMVCServerSIO.exe` の実際の場所を入力する
3. `MMVC出力デバイス名` にMMVCが書き込むデバイス名を入力する
4. OBSのOFF時／ON時デバイスを一覧から選択する
5. **MMVCを起動**、続けて **RVCを使用**

#### LANワーカーを使う

メインPC側の8771音声制御サービスは `tools/rvc/transport/` に同梱され、RVCタブから必要時に起動します。

ワーカー側PCには `tools/rvc/worker/` をフォルダごと配置し、最初に `setup_worker.cmd` を実行します。続いて `env.example` を `.env` へコピーして次を設定します。

```env
RVC_PCM_HOST=0.0.0.0
RVC_PCM_PORT=8770
RVC_PCM_TOKEN=両PCで共通の十分に長いトークン
RVC_PCM_MMVC_URL=https://127.0.0.1:18888
```

メインPCでは `data/rvc.env` を作成します。

```env
RVC_LAN_TOKEN=ワーカー側と同じトークン
```

ワーカー側は `tools\rvc\worker\start_worker.cmd` で起動し、RVCタブのホスト欄にそのPCのIPアドレスまたはホスト名を入力します。

|------|------|

| OBS WebSocket | `ws://127.0.0.1:4455` |
| RTFW API | `http://127.0.0.1:8801` |

## データの場所

アプリフォルダ直下の `data/` に保存されます。

| パス | 内容 |
|------|------|
| `data/config.json` | 基本設定・OBS など |
| `data/ui_state.json` | ウィンドウ・列幅など |
| `data/simple_comment_viewer.sqlite3` | イベント・個人設定・イベントプリセット |
| `data/icon_cache/` | ユーザーアイコン |
| `output/` | 音声・CSV など出力物 |

バックアップするなら、少なくとも `config.json` と `simple_comment_viewer.sqlite3` を残してください。

## 困ったとき

| 症状 | 確認すること |
|------|----------------|
| 起動しない | Python venv があるか。`start.cmd` のエラー表示を見る |
| コメントが来ない | lv が正しいか、放送中か、**接続** したか |
| 読み上げない | VOICEVOX 起動、基本設定の読み上げ ON、ページの「読み上げ」ON、アカウント設定で読み上げ OFF になっていないか |
| OBS に出ない | オーバーレイ URL、Browser Source、**OBS** チェック、OBS連携の接続テスト |
| リストだけ変 | 基本設定の「通常リスト」。変更後しばらく待つか OBS をリロード |
| 個人設定が効かない | 生ID（匿名 184 ではない）か、アカウントID設定で有効か |
| ログがうるさい | ログレベルを INFO に。TRACE は調査用 |
| 介入 API に繋がらない | `start.cmd` で起動したか。`GET http://127.0.0.1:8793/health` |

介入 API だけ単体で立てる場合:

```bat
start_intervention_api.cmd
```

止める:

```bat
stop_intervention_api.cmd
```

## よくある運用パターン

### A. コメント表示だけ

1. 起動
2. lv を入れて接続
3. 読み上げ OFF / OBS OFF でも可

### B. 読み上げ配信

1. VOICEVOX 起動
2. 基本設定で話者と音量
3. 接続 + 読み上げ ON
4. 常連はアカウントID設定で声を分ける

### C. OBS 付き配信

1. B に加えて OBS WebSocket と Browser Source
2. OBS連携で一括更新
3. 接続 + OBS ON
4. ギフト等はイベント設定で SE

### D. 字幕付き

1. C に加えてリアルタイム字幕でマイク開始
2. 必要なら文字起こし紐づけ

## 補足（仕組みを知りたい人向け）

- NDGR の `ChunkedMessage` を直接読み、chat 以外（ギフト・広告・通知など）も捨てずに保存します
- 取得したイベントは SQLite に残ります
- 監視アプリ本体とは別アプリです。アプリ本体とRVCのLAN音声制御コードはこのリポジトリに含まれます
- 開発者向けの設計は `DESIGN_MEMO.md` / `PLAN.md` / `AGENTS.md` を参照

以上が、使う側から見た操作の全体像です。
