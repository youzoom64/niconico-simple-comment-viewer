VOICEVOX仕様調査 - コメビュ実装向け

調査日: 2026-06-22 / 対象: VOICEVOX ENGINE OSS latest。 このメモは、Simple Comment Viewer がコメントを受け取ってからVOICEVOX音声を作るまでの内部処理を、人間が追える形にしたもの。

結論

コメビュが直接使うのはVOICEVOXエディターではなく、HTTPサーバーとして動く VOICEVOX Engine。基本は /audio_query で読み上げレシピを作り、 /synthesis でWAVを作る二段階処理。

注意点

APIの speaker は名前に反して、実際には /speakers で取れる スタイルIDを入れる。キャラ名だけでは合成できない。

このアプリでの扱い

VOICEVOXは「音声ファイルSE」ではなく、アカウントIDごとの ボイス/読み上げ設定として扱う。スキン、フォント、ボイスの同じ演出プロファイル側に置く。

VOICEVOXの全体像

エディター

人間が触るGUI。文章を入力して声を調整する表側。コメビュは通常ここを操作しない。

コメントや広告、ギフトなどを受け取る。まずraw payloadと正規化値をDBへ保存する。

種類を判定

生IDやハッシュID、event_kind から、スキン、フォント、VOICEVOXスタイル、速度設定を引く。

読み上げ文変換

URL、www、888 などを正規表現で読みやすく変換する。

再生とOBS表示

音声再生キューへ渡しつつ、OBS用HTMLにはスキンとフォント付きコメントを流す。

主要API

API 役割 このアプリでの使いどころ
GET /speakers 利用可能な話者とスタイルIDを取得する。 設定画面のVOICEVOX話者/スタイル選択に使う。speaker に渡す値はこのスタイルID。
POST /audio_query テキストからAudioQueryを作る。 コメント本文を読み上げ用の「設計図」にする入口。
POST /audio_query_from_preset プリセットを使ってAudioQueryを作る。 ユーザー別プリセット運用にする場合の候補。初期実装は通常の /audio_query でよい。
POST /synthesis AudioQueryからWAV音声を生成する。 読み上げ音声生成の本体。生成WAVは再生キューへ渡す。
POST /cancellable_synthesis キャンセル可能な音声合成。 大量コメント時に読み上げを捨てる設計にした場合の候補。エンジン側の対応確認が必要。
POST /accent_phrases テキストからアクセント句を取得する。 読み間違い修正や読み仮名制御を細かくやる時に使う。
GET /user_dict / POST /user_dict_word ユーザー辞書の参照・追加。 配信固有ワード、名前、ネットスラングの読みを安定させる。ただし辞書変更API無効化設定に注意。
GET /presets プリセット一覧を取得する。 プリセット運用を採用する場合に設定画面へ出す。

AudioQueryの中身

全体パラメータ

speedScale pitchScale intonationScale volumeScale prePhonemeLength postPhonemeLength pauseLength pauseLengthScale outputSamplingRate outputStereo kana

ここを編集すると声の速度、音高、抑揚、音量、前後の無音、ポーズ長、出力形式を変えられる。

アクセント句

accent_phrases moras accent pause_mora is_interrogative

文章を声にする前の細かい発音ブロック。将来的に読み間違いや疑問文の上げ調子を制御する時に触る。

Simple Comment Viewerでの実装方針

領域 持つ設定 処理イメージ
全体設定 VOICEVOX endpoint、timeout、worker数、デフォルト速度、キュー速度ルール。 エンジン接続先と処理能力を決める。標準は http://127.0.0.1:50021。
アカウントID設定 VOICEVOXスタイルID、速度、音量、ピッチ、スキン、フォント。 このIDがコメントした時だけ専用の見た目と声を使う。
event_kind設定 広告/ギフト/来場通知などの固定音声ファイル、表示変換テンプレート。 VOICEVOXとは別にSEを鳴らす。コメント本文があればSE中でも読み上げを走らせる。
正規表現変換 対象、pattern、replacement、有効/無効。 VOICEVOXへ渡す前にURLや草、拍手を読みやすい文字へ変換する。
キュー制御 待機数ごとの速度倍率、破棄ルール、最大同時生成数。 コメントが溜まるほど speedScale を上げる。話者IDは変えず、AudioQueryだけ調整する。

内部処理のたとえ

/audio_query は「台本を読み上げ用の楽譜に変換する工程」。 /synthesis は「その楽譜を実際に演奏してWAVにする工程」。 だから、速度や音量を変えたい時は、まず楽譜であるAudioQueryを書き換えてから合成する。

コメビュ側では、コメントを受け取った瞬間にいきなり再生しようとしない。 まずDBに保存し、イベント種類とアカウント設定を解決し、OBS表示キューとVOICEVOX生成キューへ分ける。 交通整理してから走らせる。交差点に全員を一斉突入させない。

実装時の注意

CORS

ブラウザからVOICEVOX Engineへ直接投げる場合、Origin制限に当たる可能性がある。 OBS用HTMLから直叩きせず、アプリ側バックエンド経由にする方が安定する。

辞書API

ユーザー辞書など変更系APIは、エンジン起動設定で無効化される場合がある。 辞書編集機能は失敗時の表示を必ず用意する。

WAV形式

READMEでは生成音声のサンプリングレートが24000Hzと説明されている。 再生側で扱いづらければ、アプリ内で変換または対応プレーヤーを使う。

最小実装の順番

  1. GET /speakers を呼び、話者/スタイルIDを設定画面へ出す。
  2. アカウントIDごとに style_id、speedScale、skin、font を保存できるようにする。
  3. コメント受信時に読み上げ用テキストを正規表現で変換する。
  4. POST /audio_query でAudioQueryを作る。
  5. キュー待機数に応じて speedScale を上書きする。
  6. POST /synthesis でWAVを作り、再生キューへ渡す。
  7. OBS表示は音声生成待ちで止めず、スキン/フォント付きで先に流す。
